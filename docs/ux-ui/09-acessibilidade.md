# Angeled LP — Acessibilidade

## Diretrizes

- Usar HTML semântico.
- Garantir foco visível.
- Garantir navegação por teclado.
- Manter contraste adequado no tema dark/vinho.
- Não depender apenas de cor.
- Não esconder informações importantes em imagens.

## Header e âncoras

- Links devem apontar para ids reais.
- Header fixo deve considerar `scroll-margin-top` nas seções.
- Menu mobile precisa ser acessível por teclado.

## Botões e links

- Botões para ações na página.
- Links para navegação ou URLs externas.
- CTA com texto claro.
- WhatsApp deve ter `aria-label` descritivo.

## FAQ

Obrigatório:

- Pergunta como `<button>`.
- Usar `aria-expanded`.
- Resposta associada de forma clara.
- Estado aberto/fechado perceptível sem depender apenas de ícone.

## Imagens

- Produto: alt descrevendo o Gerenciador Angeled ou placeholder.
- Mockup do app: alt informando que é preview ilustrativo, se não for screenshot real.
- OG image não substitui texto visível.

## Movimento

- Animações discretas.
- Evitar autoplay agressivo.
- Respeitar `prefers-reduced-motion` quando houver animações.

## Textos

- Tamanho mínimo confortável em mobile.
- Parágrafos com largura limitada.
- Termos legais com boa leitura e espaçamento.
